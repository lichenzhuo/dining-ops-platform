## 10. 高频面试问答

### Q1：为什么做这个学习项目？

```text
我以前项目主要是 Vue2、移动端 H5、小程序和组件库，真实业务复杂度比较高。
现在招聘更看重 Vue3、TypeScript、数据可视化、AI Coding 和复杂 B 端能力，
所以我基于自己熟悉的餐饮业务重新设计一个 Vue3 企业级运营平台，用来补齐这些技术栈。
```

### Q2：为什么不用 iframe 做报表？

```text
iframe 接入老系统成本低，适合兼容历史 BI。
但它在路由、权限、筛选、导出、样式和通信上都比较割裂。
新系统核心报表我会原生实现，只保留外部 BI 作为兼容入口。
```

### Q3：为什么用 TypeScript？

```text
这个项目有报表字段、菜单权限、路由 meta、接口 DTO、AI 生成结果等复杂结构。
架构上采用渐进式 TypeScript，核心层优先做类型约束，页面内部不过度设计。
当前实践版为 JavaScript，便于 Mock 与可视化模块快速迭代；接后端前可在 service / store 层先迁移 TS。
```

详见 [docs/10-phase-completion-audit.md](./10-phase-completion-audit.md) §5。

### Q4：ECharts 怎么封装？

```text
封装 BaseChart 统一处理初始化、销毁、loading、empty、点击事件和 resize。
容器变化用 ResizeObserver 触发 echarts.resize，避免图表变形和点击坐标异常。
```

### Q5：SVG、Canvas、ECharts 怎么选？

```text
ECharts 适合常规业务图表和大屏。
SVG 适合审批流、数据链路、营销任务流这类结构化、可交互图形。
Canvas 适合大量点位、粒子和高频重绘。
```

### Q6：D3 和 X6 的区别？

```text
D3 是数据驱动图形工具，适合高度定制的 SVG 图表。
X6 更适合流程图、关系图和编排类场景。
项目中用 D3 做自定义图表，用 X6 做审批流和营销任务流。
```

### Q7：Cesium 为什么适合这个项目？

```text
连锁餐饮有天然地理属性，例如门店分布、区域经营、订单流向和城市点位。
Cesium 可以作为高级地图态势模块，用于展示 3D 门店分布、区域柱状体和订单流向。
它不是替代后台报表，而是补充地理空间可视化能力。
```

### Q8：WebSocket 和 MQTT 怎么选？

```text
如果只是简单实时推送，用 WebSocket 就可以。
如果消息类型多，比如订单、库存、告警、导出任务、审批待办，
可以用 MQTT 的 topic 做发布订阅，降低前端消息处理复杂度。
```

### Q9：AI Agent 怎么不是玩具？

```text
我没有把 AI Agent 做成孤立聊天框，而是嵌入餐饮营销流程。
营销人员输入推广目标、关键词、平台和门店范围后，系统生成文案、图片提示词和推广方案，
再进入素材库、审批、下发门店和效果追踪，形成业务闭环。
```

### Q10：AI Coding 怎么落地？

```text
我把 AI Coding 用在需求拆解、模块边界设计、代码初稿生成、代码审查、重构优化、测试用例和文档沉淀。
不是简单让 AI 生成代码，而是每个阶段都有明确输入、边界和验收标准。
```
