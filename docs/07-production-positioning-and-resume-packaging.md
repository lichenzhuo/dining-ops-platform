# 真实业务级包装与简历口径

## 1. 结论

如果希望把 Dining Ops Platform 包装成“真实项目”，建议使用更准确的说法：

```text
基于餐饮连锁真实业务场景设计并实现的 B 端运营平台
```

而不是：

```text
某公司线上生产项目
```

这样既能体现项目真实业务价值，又避免虚构工作经历。

## 2. 对外推荐口径

推荐叫法：

- 独立项目
- 作品项目
- 技术实践项目
- 业务级项目
- 生产级设计项目

推荐项目描述：

```text
Dining Ops Platform 是一个基于餐饮连锁企业经营、营销、报表和门店执行场景设计的 B 端运营平台，
覆盖经营工作台、原生报表中心、经营指挥大屏、地图可视化、AI 营销 Agent、审批中心和数据导入导出等模块。
项目按照真实企业后台系统的交互、权限、报表、性能和可扩展性要求进行设计与实现。
```

简历标题可写：

```text
个人项目：餐饮连锁运营平台 Dining Ops Platform
```

或者：

```text
独立项目：餐饮连锁运营平台 Dining Ops Platform
```

如果在线简历不适合写“个人项目”，可以写：

```text
项目作品：餐饮连锁运营平台
```

## 3. 不建议的口径

不要写：

- “负责某餐饮公司线上运营平台开发”
- “服务全国数百家门店”
- “上线后提升营收 xx%”
- “公司内部生产系统”
- “真实接入美团、抖音、POS、CRM 生产接口”

除非这些确实发生且可解释。

可以写：

- “模拟 POS、外卖、CRM、库存、财务等数据源”
- “保留真实企业系统的数据接入扩展点”
- “按生产级系统设计权限、报表、导入导出和性能优化方案”
- “基于过往餐饮业务经验抽象业务模型”

## 5. 简历项目写法

### 项目描述

```text
餐饮连锁运营平台 Dining Ops Platform：
基于 Vue3、TypeScript 和 Element Plus 设计并实现的 B 端运营平台，
面向连锁餐饮总部、区域经理、门店店长和营销人员，覆盖经营工作台、原生报表中心、经营大屏、地图可视化、AI 营销 Agent、审批中心和数据导入导出等模块。
项目基于餐饮连锁真实业务场景抽象，模拟 POS、外卖、CRM、库存和财务等数据源，形成经营分析、报表下钻、AI 营销、审批下发和效果追踪闭环。
```

### 核心贡献

- 搭建 Vue3 + Vite + TypeScript + Element Plus 中后台工程，设计多布局、动态路由、Pinia 状态持久化和权限菜单。
- 设计原生报表中心，支持报表分类、统一筛选、字段配置、保存视图、下钻分析、服务端分页、虚拟表格和异步导出任务。
- 封装 SearchForm、DataTable、BaseChart、MetricCard、PermissionButton 等业务组件，统一处理查询、表格、图表 resize、权限和异常状态。
- 基于 ECharts 实现经营大屏、趋势分析、渠道占比、区域热力、门店点位和订单飞线。
- 建设可视化实验室，使用 SVG、Canvas、D3、X6、Three.js、Cesium 补充流程、关系、3D 地图和大量数据可视化能力。
- 设计 AI 营销 Agent，将推广目标、关键词、平台规则和门店范围转化为文案、图片提示词和推广任务，并串联素材库、审批、下发门店和效果追踪。
- 使用 WebSocket / MQTT 模拟实时订单、库存预警、审批待办、导出任务和数据链路状态。
- 沉淀 AI Coding 工作流，用于需求拆分、模块设计、代码生成、重构优化、测试和文档沉淀。

## 6. 面试被问“这是个人项目还是真实项目”怎么答

推荐回答：

```text
这是我基于过往餐饮连锁业务经验做的独立项目，不是简单 Demo。
业务模型来源于我之前做过的点餐、CRM、门店运营、报表和营销推广场景，
但数据是脱敏和模拟的，没有使用原公司的真实数据。
我按照真实 B 端系统标准做了权限、报表、导入导出、大屏、地图、AI Agent 和性能优化设计。
```

如果面试官追问“有没有线上用户”：

```text
这个项目目前主要作为作品项目和技术实践项目，没有接真实商户生产数据。
但我在之前公司做过真实餐饮连锁业务系统，包括点餐小程序、CRM 营销系统、PC 管理平台和组件库。
这个项目是把之前业务经验迁移到 Vue3、TypeScript、可视化和 AI Agent 技术栈上的系统化实践。
```

## 7. 对外 README 和内部文档分工

对外 README：

- 强调产品定位。
- 强调技术架构。
- 展示截图。
- 说明本地运行。
- 说明核心功能。
- 不写“学习路线”“补短板”等内部表达。

内部 docs：

- 保留学习路线。
- 保留面试准备。
- 保留 AI 使用方式。
- 保留招聘对齐分析。
